Local environment construction

Such a construction can be divided into following steps:

  1. Install nodejs
  2. Git is necessary
  3. Create a fold in a directory you prefer and then, under this directory, use commands in cmd (yes, the command line of windows)

Installation and Usage

What should be emphasized here is that the name of configuration file should be modified to the correct form _config.yml. Update

It is better to backup your _config.yml file of the theme before starting update.

Deploy to Github pages

A special repository whose name is ‘user-name.github.io’ should be firstly created on Github and set the Github pages selections. Then, it is time to deploy the Hexo static blog to Github. Note that our construction must be done with in this folder. Otherwise, you cannot use Git to deploy your blog to Github pages.

Start your writing

Command hexo n will automatically create a .md file in ./source/_posts. Your writing is just in those .md files. Actually, once the blog constructed, you just need create .md files for your posts.

A new page can be created by hexo new page <page-name>.
